Generic

For all IO monitoring module, these are common for them all.

Name Type Required Default Value Description
directory java.io.File YES   The root local directory. Must be a valid directory where you can read and write.
fileProcessor dk.heick.io.monitoring.processor.FileProcessor YES   An implementation of the FileProcessor interface.
stableTime long no 1500 (1.5 seconds) How many milliseconds a file has be stable before we deem it save to process.
archiving boolean no true Should a file be archived (true) or deleted (false) after successful processing.
maxMessagesPolling int no 1000 The max number of files added to the file monitor before continue to the next to phases (monitoring and processing).
This is to avoid that is added huge number files (which can be slow) before start processing.
If below "1" than no limit.
maxProcessingTime long no 30000 (30 seconds) The maximum amout of time the processing phase must take before existing and waiting for another "runTask" execution.
This to avoid a that the "processing" step takes a long time before we start added and monitoring new files.
If below "1" than no limit.
monitorTimeout long no 3600000 (1 hour) The maximum amount of time we will monitor a file before we deem it dead (has not been stable in the amount of time), and move it to ".error" directory.
If below "1" than no limit.
